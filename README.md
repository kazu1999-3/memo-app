# memo-app

A simple CLI-based memo application built with TypeScript.

## Features

- Create, read, update, and delete memos
- Tag-based organization
- Full-text search across titles, content, and tags
- JSON-based file storage
- TypeScript with strict mode
- Comprehensive test coverage

## Getting Started

### Prerequisites

- Node.js 20 or higher
- npm

### Installation

```bash
npm install
```

### Usage

#### Add a new memo
```bash
npm run dev add "Meeting Notes" "Discuss project timeline" work,meeting
```

#### List all memos
```bash
npm run dev list
```

#### List memos by tag
```bash
npm run dev list --tag work
```

#### Show a specific memo
```bash
npm run dev show <memo-id>
```

#### Search memos
```bash
npm run dev search "project"
```

#### Update a memo
```bash
npm run dev update <memo-id> --title "New Title" --content "New content"
```

#### Delete a memo
```bash
npm run dev delete <memo-id>
```

#### Show help
```bash
npm run dev help
```

### Development

```bash
npm run dev          # Run the CLI
npm run build        # Build project
npm test             # Run tests
npm run typecheck    # Check types
npm run lint         # Lint code
```

1. **CoordinatorAgent** - Task planning & orchestration

## Configuration

### Environment Variables

Required variables (see `.env.example`):

- `GITHUB_TOKEN` - GitHub personal access token
- `ANTHROPIC_API_KEY` - Claude API key (optional for local development)
- `REPOSITORY` - Format: `owner/repo`

### GitHub Actions

Workflows are pre-configured in `.github/workflows/`:

- CI/CD pipeline
- Automated testing
- Deployment automation
- Agent execution triggers

**Note**: Set repository secrets at:
`https://github.com/kazu1999-3/memo-app/settings/secrets/actions`

Required secrets:
- `GITHUB_TOKEN` (auto-provided by GitHub Actions)
- `ANTHROPIC_API_KEY` (add manually for agent execution)
